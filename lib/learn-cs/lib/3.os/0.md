# 操作系统特性

并发：
宏观上看上去同时发生。

并行：
多核心处理。
多个事件同时发生

共享：
资源共享

虚拟：
将物理实体变为若干个逻辑上的对应物
例如，多个应用程序同时运行，看上去有多个CPU，但是其实只有一个。

异步：
多个程序可以并发执行，但资源有限，进程执行受到其他因素影响不可预知结束时间。

分时操作系统：
以时间片为单位轮流为各个用户服务。
对于用户是平等的，不区分优先级。

实时操作系统：
增加优先的紧急任务，有些紧急任务不需要时间片排队。并且紧急任务必须被在严格的事件限制内处理完（硬实时系统）。如果没有对紧急任务处理事件做严格限定，则为软实时系统。
